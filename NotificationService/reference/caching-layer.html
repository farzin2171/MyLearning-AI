IServiceCacheManager Interface

Task<T>   GetAsync<T>(string key)                              // read
Task      SetAsync<T>(string key, T value, TimeSpan timeToLive) // write

Implemented by NotificationServiceCacheManager (singleton). Called by KeepAliveManagementService.

GetAsync — Read Path

if (circuit is Open)  → FallbackToInMemoryCache(key)      // early exit
try {
    ExecuteAsync → _redisCache.GetStringAsync(key)         // Redis
    Deserialize JSON → return T
}
catch { FallbackToInMemoryCache(key) }                     // any Redis error

SetAsync — Write Path

if (circuit is Open)  → FallbackToInMemoryCache(key, value)  // early exit
try {
    ExecuteAsync → {
        AcquireLock(key + "_lock")       // RedLock distributed lock
        if (!lock.IsAcquired) throw      // lock fail = circuit breaker hit
        AddStringAsync(key, json, ttl)   // write to Redis
    }
}
catch { FallbackToInMemoryCache(key, value) }               // any error

Lock prevents two instances overwriting the same key simultaneously. Lock key = "{key}_lock".

Circuit Breaker States

StateHow you enterWhat happensHow you exit
Closed Normal / after successful probe Redis is tried for every call N exceptions → Open
Open N exceptions in Closed state Redis is bypassed; in-memory cache used; OnCircuitBreak logged Break duration expires → HalfOpen
HalfOpen Break duration expires (default: 60s) One probe request allowed through to Redis Probe succeeds → Closed; probe fails → Open again

Fallback Caveat

The in-memory fallback is per-instance. In a multi-replica deployment, instance A's memory cache is invisible to instance B. During a Redis outage, keep-alive signals may not find connections registered on a different instance.
