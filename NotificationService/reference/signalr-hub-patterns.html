SignalR Hub Patterns

Quick-reference for hub anatomy, lifecycle, and client targeting in the Notification Service

Hub Class Anatomy

// Every hub inherits from Hub (or Hub<T> for typed clients)
[Authorize]
public class NotificationHub : Hub
{
    // Dependencies injected normally — a new instance per connection call
    public NotificationHub(IKeepAliveManagementService svc) { … }

    // ── Lifecycle (called by SignalR automatically) ──────────
    public override async Task OnConnectedAsync()    { … }
    public override async Task OnDisconnectedAsync(Exception? ex) { … }

    // ── Hub methods (client → server, invoked by browser) ───
    public async Task SetApplicationId(Guid appId)  { … }
    public async Task UnSetApplicationId(Guid appId){ … }
}

Connection Context Properties

PropertyTypeWhat it contains
Context.ConnectionIdstringUnique ID assigned by SignalR to this WebSocket socket. Changes on every reconnect.
Context.UserIdentifierstring?The user ID extracted from the JWT (maps to the NameIdentifier claim). Used by .User(userId) routing.
Context.UserClaimsPrincipalThe full authenticated principal — all JWT claims available here.

Lifecycle Methods

MethodWhen it firesWhat this service does
OnConnectedAsync()WebSocket handshake completeAdds connection to "Authenticated SignalR Users" group
OnDisconnectedAsync(ex)WebSocket closes (clean or error)Removes connection from the group. ex is null on clean disconnect.

Hub Methods — Client Calls Server

MethodCalled whenWhat it does
SetApplicationId(appId)Browser tab registers on loadStores {appId → connectionId} in cache via KeepAliveManagementService.LinkAsync
UnSetApplicationId(appId)Browser tab unloadsRemoves the mapping via KeepAliveManagementService.UnlinkAsync

Client Targeting — Server Calls Client

MethodTargetsUsed for
Clients.User(userId) notifications All connections whose Context.UserIdentifier matches. Managed by SignalR's Redis backplane. NotificationsController — sends notificationReceived
Clients.Clients(ids) keep-alive A specific list of ConnectionId strings. List comes from the application's own Redis cache. KeepAliveController — sends keepAliveReceived
Clients.Group(name) All connections added to the named group via Groups.AddToGroupAsync. Available (e.g. broadcast to "Authenticated SignalR Users") — not actively used for delivery in this service
Clients.All Every connected client, authenticated or not. Not used in this service (it requires [Authorize])

IHubContext — Sending from Outside the Hub

// Inject into a controller or service:
public NotificationsController(IHubContext<NotificationHub> hubContext)
{
    _hubContext = hubContext;
}

// Then push messages from anywhere:
await _hubContext.Clients.User(userId).SendAsync("notificationReceived", payload);
await _hubContext.Clients.Clients(ids).SendAsync("keepAliveReceived");

IHubContext<T> exists because a Hub instance is created per connection call — it cannot be held as a long-lived dependency. Controllers and services use IHubContext to reach clients without being inside a hub class.
