Orchestration Rules (ExternalRolesService)

Role Name → Access Key Mapping (two levels)

Provider.Name (string) → tblExternalRole.ExternalSourceId (must match exactly, case-sensitive) → tblExternalRoleToRole → tblRole → tblPermission → tblExternalRoleToPermission → tblPermission → [dbo].[GetUserAccessKeys] returns Permission.Key strings

"Why Is This User Denied?" Checklist

  1. Check logs for provider exceptions (LogError from ExternalRolesService)
  2. Confirm providers are IsActive = 1 in tblExternalRoleProvider
  3. Confirm role name matches tblExternalRole.ExternalSourceId exactly (case!)
  4. Confirm ExternalRole has active mappings in tblExternalRoleToRole / tblExternalRoleToPermission
  5. Clear stale cache: DELETE /api/v1/cache/permissions/{userId}
