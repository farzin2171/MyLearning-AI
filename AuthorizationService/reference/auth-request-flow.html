Fast-path vs Slow-path

Cache HIT → keysSet.Contains(key) → Allow/Deny (no DB, no ext calls) Cache MISS → ExternalRoles → [dbo].[GetUserAccessKeys] → SetAsync → Contains

Cache Resilience

StateBehaviour
Redis healthyNormal read/write. Lock key = {cacheKey}_lock prevents stampede.
Redis degraded (breaker closed, errors accumulating)Polly retries; errors counted toward breaker threshold.
Redis down (breaker open)All reads/writes go to per-instance in-memory cache. Invalidation doesn't propagate.

OnBehalfOf — Extra Steps

  1. Resolve external roles for impersonator.
  2. Check impersonator has "authorization.use.onBehalfOf" permission. If not → return empty (Deny).
  3. Resolve external roles for target user.
  4. Run [dbo].[GetUserAccessKeys] for target user's roles.
  5. If impersonator is an assistant → run [dbo].[GetUserAccessKeysFromPermissionGroups] and intersect results.
  6. Cache intersected list under composite key.
