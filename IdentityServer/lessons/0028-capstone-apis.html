Capstone — Build the Calendar API & Reports Worker

~40 minutes Hands-on build Prerequisite: Lesson 27 — WorkSpace IS running on port 5001

WorkSpace IS is up and issuing tokens. This lesson builds two more projects: the Calendar API that validates those tokens and enforces tenant isolation, and the Reports Worker that acquires its own service token to call the API.

3

Wire JWT Bearer and scope policies in Program.cs

Program.cs
var builder = WebApplication.CreateBuilder(args);

builder.Services
    .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.Authority = "https://localhost:5001";

        // Duende IS sets aud to the scope name when no ApiResource groups it
        options.TokenValidationParameters.ValidateAudience = false;

        // Allow self-signed cert from IS in development
        options.BackchannelHttpHandler = new HttpClientHandler
        {
            ServerCertificateCustomValidationCallback = (_, _, _, _) => true
        };
    });

builder.Services.AddAuthorization(options =>
{
    options.AddPolicy("CanReadCalendar",  p => p.RequireClaim("scope", "calendar.read"));
    options.AddPolicy("CanWriteCalendar", p => p.RequireClaim("scope", "calendar.write"));
});

// Register the ownership requirement handler
builder.Services.AddSingleton<IAuthorizationHandler, EventOwnerHandler>();

builder.Services.AddControllers();

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();
app.Run();
ValidateAudience = false This is a development shortcut. In production, configure ApiResource in Config.cs and set ValidAudience to the resource name. Duende IS then stamps aud on the token automatically.
4

Implement the EventOwner requirement and handler

Create Authorization/EventOwnerRequirement.cs:

Authorization/EventOwnerRequirement.cs
public class EventOwnerRequirement : IAuthorizationRequirement { }

public class EventOwnerHandler
    : AuthorizationHandler<EventOwnerRequirement, CalendarEvent>
{
    protected override Task HandleRequirementAsync(
        AuthorizationHandlerContext context,
        EventOwnerRequirement requirement,
        CalendarEvent resource)
    {
        var tokenTenantId = context.User.FindFirstValue("tenant_id");

        if (tokenTenantId != null
            && tokenTenantId == resource.TenantId)
        {
            context.Succeed(requirement);
        }

        return Task.CompletedTask;
    }
}
5

Create the CalendarController

Controllers/CalendarController.cs
[ApiController]
[Route("api/[controller]")]
[Authorize(Policy = "CanReadCalendar")]   // all actions need calendar.read
public class CalendarController : ControllerBase
{
    private readonly IAuthorizationService _authz;

    public CalendarController(IAuthorizationService authz) => _authz = authz;

    // GET api/calendar — return events for the caller's tenant only
    [HttpGet]
    public IActionResult GetEvents()
    {
        var tenantId = User.FindFirstValue("tenant_id");
        var events   = EventStore.Events
            .Where(e => e.TenantId == tenantId)
            .ToList();

        return Ok(events);
    }

    // DELETE api/calendar/{id} — must own the event
    [HttpDelete("{id:int}")]
    [Authorize(Policy = "CanWriteCalendar")]   // also needs write scope
    public async Task<IActionResult> DeleteEvent(int id)
    {
        var calEvent = EventStore.Events.FirstOrDefault(e => e.Id == id);
        if (calEvent == null) return NotFound();

        // Imperative: check ownership against the specific event
        var result = await _authz.AuthorizeAsync(User, calEvent, new EventOwnerRequirement());
        if (!result.Succeeded) return Forbid();

        EventStore.Events.Remove(calEvent);
        return NoContent();
    }
}
Verify → dotnet build succeeds, then dotnet run starts the API at https://localhost:5003.

Expected HTTP responses

RequestTokenExpected
GET /api/calendar None 401 — no token
GET /api/calendar Alice's token with calendar.read 200 — ACME events only
GET /api/calendar Bob's token with calendar.read 200 — Globex events only
DELETE /api/calendar/3 Alice's token (tenant-acme, event is tenant-globex) 403 — wrong tenant
DELETE /api/calendar/1 Alice's token (tenant-acme, event is tenant-acme) 204 — deleted

Wire token management and a named HttpClient in Program.cs

Program.cs
var builder = Host.CreateApplicationBuilder(args);

// Token management — declare the IS token endpoint and client credentials
builder.Services.AddClientCredentialsTokenManagement()
    .AddClient("calendar-api", client =>
    {
        client.TokenEndpoint = "https://localhost:5001/connect/token";
        client.ClientId      = "reports-worker";
        client.ClientSecret  = "worker-secret";
        client.Scope         = "calendar.read";
    });

// Named HttpClient — automatically attaches a valid Bearer token
builder.Services.AddClientCredentialsHttpClient(
    httpClientName: "CalendarClient",
    tokenClientName: "calendar-api",
    configureClient: client =>
    {
        client.BaseAddress = new Uri("https://localhost:5003");

        // Trust dev cert from Calendar API
        // In production: use proper cert validation
    });

builder.Services.AddHostedService<ReportsWorker>();

var host = builder.Build();
host.Run();
8

Implement the worker

ReportsWorker.cs
public class ReportsWorker : BackgroundService
{
    private readonly IHttpClientFactory _factory;
    private readonly ILogger<ReportsWorker> _logger;

    public ReportsWorker(
        IHttpClientFactory factory,
        ILogger<ReportsWorker> logger)
    {
        _factory = factory;
        _logger  = logger;
    }

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            try
            {
                // Token is automatically acquired, cached, and renewed
                var client   = _factory.CreateClient("CalendarClient");
                var response = await client.GetAsync("/api/calendar", stoppingToken);

                if (response.IsSuccessStatusCode)
                {
                    var json = await response.Content.ReadAsStringAsync(stoppingToken);
                    _logger.LogInformation("Calendar data: {data}", json);
                }
                else
                {
                    _logger.LogWarning("Unexpected status: {status}", response.StatusCode);
                }
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "Report run failed");
            }

            await Task.Delay(TimeSpan.FromSeconds(30), stoppingToken);
        }
    }
}
Verify → Start the worker with dotnet run (CalendarApi must also be running). Watch the console — you should see "Calendar data: […]" logged every 30 seconds. The worker fetches a token once and reuses it until near expiry.
