Authorization Service — Quick Reference

Print-friendlyServices.Authorization

Endpoints

GET /api/v1/Authorize?key={accessKey} Auth: any valid JWT (user or service) · 200 = allowed, 403 = denied
GET /api/v1/Evaluate Auth: service account only (service_isService: true) · Returns full access-key array
GET /api/v1/Evaluate/{userId}?productKey={key} Auth: service account only · Returns boolean
DELETE /api/v1/Cache/permissions/{executedBy}?executedFor={executedFor} Auth: service account + IsService policy · Immediate cache invalidation
DELETE /api/v1/Cache/clear/assistants/{assistantId} Auth: any authenticated user · Called by AssistantManagement on relation changes

Useful SQL Queries

-- Manually compute access keys for debugging:
EXEC spGetUserAccessKeys @TenantId='…', @Roles='Advisor'

-- Find all permissions for a policy:
SELECT perm.Key
FROM tblPermission perm
JOIN tblPolicyPermissionMapping ppm ON ppm.PermissionId = perm.PermissionId
JOIN tblPolicy p ON p.PolicyId = ppm.PolicyId
WHERE p.Key = 'eapp' AND perm.IsActive = 1

-- Find which policies a role has access to:
SELECT p.Key AS Policy
FROM tblPolicyRoleMapping prm
JOIN tblPolicy p ON p.PolicyId = prm.PolicyId
JOIN tblRole   r ON r.RoleId   = prm.RoleId
WHERE r.Key = 'Advisor'
