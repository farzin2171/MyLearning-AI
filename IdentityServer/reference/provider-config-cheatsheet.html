"Properties": { "EcosystemTenant": "ACME", req "Authority": "https://org.okta.com/oauth2/default", req OIDC issuer URL "ClientId": "0oa1b2c3...", req "ClientSecret": "...", req "Scopes": "openid profile email", opt "CallbackPath": "/signin-okta", opt register in Okta as redirect URI "GetClaimsFromUserInfoEndpoint": "true", opt default: true "ClaimsToRemove": ["ver","at_hash","nonce"], opt strip provider-specific noise "ApplyUrlDecode": "false" opt true if state is URL-encoded } // CRITICAL: Add any claims starting with "client_" or "service_" to ClaimsToRemove // or the IdG will THROW and reject the login

Post-Login Callback Flow

#StepCode location
1Provider redirects to /External/CallbackExternalController.cs
2Extract sub or nameidentifier as user IDFindUserFromExternalProviderAsync()
3oid → externalobjectidentifier claim remapCustomClaimTypes.OutboundClaimTypeMap
4EcosystemTenant → tenant claim addedTenantAccessor.GetAsync(scheme)
5User looked up or auto-provisioned in UserDbUserStore.FindByExternalProviderAsync()
6External session stored in RedisPopulateUserSessionInformationAsync()

Forbidden Claims (Will Throw)

Claims from any external provider whose type starts with client_ or service_ cause a hard exception — unless the claim is in the allow-list (tenantId, identityRole). Add them to ClaimsToRemove.
