Glossary

Quick reference Updated as lessons progress

Canonical definitions for all terms used across the Equisoft Identity learning course. When a lesson uses a term, it matches the definition here.

ACDE GIJO PRST W

A

Access Key
A dot-separated string that represents a single permission in the Authorization Service's policy hierarchy. Format: policy.subpolicy.permissionOrRole. Example: shared.eapp.Lock.
The GET /api/v1/Authorize?key=… endpoint returns Allow/Deny for a given access key. Source: Services.Authorization — AuthorizationConstants.cs, spGetUserAccessKeys.sql
Access Token
A short-lived JWT presented to APIs in the Authorization: Bearer … header. Contains audience, scopes, subject, and Equisoft custom claims (tenantId, identityRole, etc.). Not to be confused with the ID Token. Source: RFC 6749, RFC 9068
Authorization Code Flow
The OAuth 2.0 grant type used by user-facing applications. The client receives a short-lived code via the browser redirect, then exchanges it for tokens server-side. Always used with PKCE in the Equisoft ecosystem. Source: RFC 6749 §4.1, Lesson 2
Authorization Service
Services.Authorization. The Equisoft microservice that translates a user's roles into a flat set of access keys using a policy hierarchy stored in SQL. It does not issue tokens; it only answers "does this user have permission X?". Source: Lesson 1

C

Client Credentials Grant
The OAuth 2.0 grant type for service-to-service calls. The service authenticates with its own client_id and client_secret. Tokens issued do not contain a sub claim (no user subject) but do contain service_isService: true. Source: RFC 6749 §4.4, Lesson 2
Claims
Key-value pairs in a JWT payload that assert facts about the user or the token itself. Standard claims: sub, iss, aud, exp. Equisoft custom claims: tenantId, tenant, identityRole, externalobjectidentifier, isGuest, service_isService. Source: RFC 7519, Lesson 3

D

Duende IdentityServer
The commercial .NET library used to build the Identity Gateway. Version 7.3.2 in the Equisoft ecosystem. It implements OAuth 2.0 and OpenID Connect. The Identity Gateway extends it with custom validators, token generators, and profile service implementations. Source: docs.duendesoftware.com
Discovery Endpoint
The /.well-known/openid-configuration URL on the Identity Gateway. Returns the JWKS URI, token endpoint, authorization endpoint, and supported grant types. API services use it to fetch signing keys for JWT validation. Never hard-code signing keys — use discovery. Source: OpenID Connect Discovery 1.0

E

EquisoftProfileService
Custom Duende IS IProfileService implementation in the Identity Gateway. Loads user claims from the local UserDb (name, email, external provider claims) and filters which extra claims are included in tokens. Source: IdentityGateway/IdentityServer/Services/EquisoftProfileService.cs.
EquisoftTokenResponseGenerator
Custom component that fires after the standard token response is built. Makes outbound HTTP calls to the Tenant Service and User Service to enrich tokens with tenantId and identityRole. Source: IdentityGateway/IdentityServer/Services/EquisoftTokenResponseGenerator.cs.
External Role Provider
A pluggable component in the Authorization Service that fetches a user's roles from an external source. Types: ClaimBased, MicrosoftGraph, CustomApi, OipaExposedComputation, Assistant. Configured per tenant in the Authorization Service database. Source: Services.Authorization — ExternalRole/Providers/
externalobjectidentifier
Equisoft custom JWT claim. Holds the user's identifier in the external identity provider (e.g. the Azure AD Object ID). Mapped from the IdP's oid claim during login. Used by the User Service and Authorization Service to look up the user's role. Source: Lesson 3

G

Grant Type
The OAuth 2.0 mechanism by which a client obtains tokens. The three grant types used in Equisoft: Authorization Code + PKCE (user login), Client Credentials (service-to-service), and JWT Bearer Assertion (federated SSO). See Lesson 2.

I

ID Token
A JWT issued alongside the access token in the Authorization Code flow. Intended for the client application — not for APIs. Contains user identity information (name, email, sub). Never send the ID Token in an API's Authorization header. Source: OpenID Connect Core 1.0 §2
Identity Gateway (IdG)
Applications.IdentityGateway. The central OAuth 2.0 / OpenID Connect provider for the Equisoft platform, built on Duende IdentityServer v7.3.2. Issues access tokens, ID tokens, and refresh tokens. Authenticates users via local accounts and external providers. Source: Lesson 1
identityRole
Equisoft custom JWT claim. The user's platform role (e.g. Advisor, Admin, Guest). Added to tokens by EquisoftTokenResponseGenerator via a call to the User Service. Guest users get this set to "Guest" without an external call. Source: Lesson 3
IIdentityContext
DIT (Digital Insurance Tools) framework interface. Available via DI in ASP.NET Core services. Parsed from the incoming JWT. Key properties: TenantId, ResourceOwnerId (sub), ExternalPrincipalId, IdentityType, IsOnBehalfOf. Source: AssistantManagement + Authorization services — IdentityContextExtensions.cs
isGuest
Equisoft custom JWT claim. Present and "true" on guest user tokens. Used by EquisoftTokenResponseGenerator to short-circuit role lookup and set identityRole = "Guest". Source: Lesson 3

J

JWT Bearer Assertion Grant
Custom grant type (urn:ietf:params:oauth:grant-type:jwt-bearer) used for federated SSO. A trusted external system presents a signed JWT asserting a user's identity. The Identity Gateway validates the signature and issues a standard Equisoft access token. Defined in RFC 7523. Source: RFC 7523, JwtBearerAuthorizeAssertionExtensionGrantValidator.cs, Lesson 2
JWKS (JSON Web Key Set)
The set of public keys published by the Identity Gateway at its JWKS URI (found via discovery). API services use these keys to verify the signature on incoming JWTs without needing to call the Identity Gateway on every request.

O

OpenID Connect (OIDC)
An identity layer on top of OAuth 2.0. Adds the concept of an ID Token, UserInfo endpoint, and discovery document. The Identity Gateway implements OIDC on top of Duende IdentityServer. Source: OpenID Connect Core 1.0

P

Permission
The most granular unit in the Authorization Service's hierarchy. A permission has a Key (string) and can be assigned directly to a Policy. Example: Lock. Combined with the policy path to form an Access Key: shared.eapp.Lock. Source: Services.Authorization — Permission.cs
PKCE (Proof Key for Code Exchange)
A security extension to the Authorization Code flow (RFC 7636). The client generates a code_verifier, hashes it to a code_challenge, and sends the challenge with the authorization request. The verifier is sent with the token request. Prevents authorization code interception attacks. Mandatory for all public clients in Equisoft.
Policy
A node in the Authorization Service's hierarchical permission model. Policies can be nested (parent/child). A policy contains roles and permissions. Access keys are formed by concatenating the policy path with a permission or role key using . as separator. Source: Services.Authorization — Policy.cs, spGetUserAccessKeys.sql

R

Refresh Token
A long-lived credential used to obtain new access tokens without re-authenticating the user. In the Identity Gateway, refresh tokens are stored in Redis alongside the external IdP session expiry. If the external session expires, the refresh is rejected even if the refresh token itself is still valid. Source: Lesson 2, EquisoftTokenRequestValidator.cs
Role
A named group in the Authorization Service. Roles are mapped to Policies. A user's identityRole claim (or external role from a provider) is resolved to an internal Role, which is then used to compute the user's access keys. Source: Services.Authorization — Role.cs

S

Scope
An OAuth 2.0 concept that limits what a token can access. In the Equisoft ecosystem, scopes represent API resources and fine-grained permissions (e.g. assistantManagement.create, write:assistantManagement). A client must request the right scopes; the user (or admin) must have consented.
service_isService
Equisoft custom JWT claim. Present only on service-account tokens (Client Credentials grant). The value is "true". The DIT Identity framework reads this to set IdentityType.Service. The Authorization Service's IsService policy requires this claim on management endpoints. Source: Lesson 3

T

tenant
Equisoft custom JWT claim. Short human-readable key for the user's tenant (e.g. ACME). Used for service account secrets lookup in config (TenantSecrets) and log correlation. Different from tenantId (GUID). Loaded from the UserProvider table by EquisoftProfileService. Source: Lesson 3
tenantId
Equisoft custom JWT claim. GUID primary key of the user's tenant in the platform database. Used by downstream services as the foreign key for data isolation (every query filtered by _identityContext.TenantId). Added by EquisoftTokenResponseGenerator via a call to the Tenant Service. Source: Lesson 3

W

Wildcard Redirect URI / CORS
Equisoft extensions to standard Duende IS that allow clients to register redirect URIs and CORS origins with wildcards (e.g. https://*.equisoft.com/*). Implemented by WildcardRedirectUriValidator and WildcardCorsPolicyService in the Identity Gateway. Standard Duende IS requires exact matches.
