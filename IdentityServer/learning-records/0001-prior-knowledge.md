# User has working knowledge of OAuth 2.0 and OpenID Connect

The user already understands OAuth 2.0 and OpenID Connect conceptually. This means we can skip introductory definitions of flows, tokens, and claims — and instead dive directly into how these protocols are applied in the Equisoft ecosystem.

**Evidence:** Stated directly: "I know about OpenId connect and Oauth."

**Implications:**
- Do not re-teach what an access token or ID token is from scratch
- Do connect every lesson to real code in the three services
- Lessons should start at "here's how we use it" rather than "here's what it is"
- The zone of proximal development starts at: mapping protocol knowledge to implementation patterns in a multi-service architecture
