Gotchas

Rule
Build jobs must use fetchDepth: 0 — full history required for tag lookup
PR validation jobs can keep fetchDepth: 1 (no GitVersion)
gitversion/setup must run before gitversion/execute
Variables are only available to steps after execute runs
Config lives in GitVersion.yml at repo root

Pipeline YAML snippet

- checkout: self
  fetchDepth: 0                    # required — full history

- task: gitversion/setup@3.2.1
  displayName: Setup GitVersion
  inputs:
    versionSpec: '6.x'

- task: gitversion/execute@3.2.1
  displayName: Execute GitVersion
  inputs:
    useConfigFile: true
    configFilePath: GitVersion.yml

# Variables now available:
# $(GitVersion.SemVer)         → use for Octopus ReleaseNumber, npm version
# $(GitVersion.MajorMinorPatch) → stable number without pre-release label
# $(GitVersion.AssemblySemVer) → .NET assembly version (four-part)
